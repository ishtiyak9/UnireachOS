Detailed Findings

Path Traversal (Arbitrary File Write) in Document Uploads

Critical

Location: server/api/applicants/[id]/documents.post.ts

Description: The document upload endpoint improperly sanitizes the uploaded file's original name before using it in filesystem operations. By crafting a multipart payload with a filename containing directory traversal sequences (e.g., ../../../evil.js), an attacker can write files to arbitrary locations on the server, potentially overwriting critical system files or planting executing scripts (Remote Code Execution).

// Vulnerable Code const fileName = `${Date.now()}-${file.filename}`; const filePath = path.join(applicantDir, fileName); await fs.writeFile(filePath, file.data);

Remediation

Sanitize the incoming filename completely. Strip pathway characters, or preferably, generate a sterile UUID for local filesystem storage to completely abstract the user's input from the filesystem constraints.

import { randomUUID } from "node:crypto"; // Recommended const ext = path.extname(file.filename || ''); const fileName = `${Date.now()}-${randomUUID()}${ext}`;

Vulnerable Dependencies (CVE-2026-28292)

Critical

Location: pnpm-lock.yaml / package.json

Description: PNPM audit tools revealed a critical vulnerability in the simple-git dependency module (CVE-2026-28292). A blockUnsafeOperationsPlugin bypass via a case-insensitive protocol.allow config key exists which enables Remote Code Execution (RCE).

Remediation

Run dependency upgrades and update simple-git to version >=3.32.3. Audit the dependency tree utilizing pnpm update --interactive --latest.

Global SSL/TLS Verification Disabled

High

Location: .env

Description: The application environment enforces NODE_TLS_REJECT_UNAUTHORIZED=0. This effectively instructs the Node.js runtime to blindly accept any inherently untrusted, expired, or self-signed SSL certificates. This leaves outgoing requests—and potentially database connections (Aiven PostgreSQL is used)—open to severe Man-In-The-Middle (MitM) attacks.

Remediation

Remove NODE_TLS_REJECT_UNAUTHORIZED=0 from the `.env` settings. If the environment requires custom/local certificates, install self-signed certificates at the OS/Node trust store bounds rather than disabling validation entirely.

Minimatch ReDoS Vulnerability

High

Location: pnpm-lock.yaml

Description: The project relies on deep versions of minimatch (< 3.1.3, < 9.0.7) that suffer from a Regular Expression Denial of Service (ReDoS) vulnerability (GHSA-7r86-cg39-jmmj). Combinatorial backtracking can be exploited to crash services.

Remediation

Use PNPM overrides or resolutions to force minimatch into safe, patched versions.

Permissive Security Headers (CORS & CSP)

Medium

Location: nuxt.config.ts

Description: Nuxt Security module sets a wildcard CORS handler (origin: "*") and enforces contentSecurityPolicy: false. While noted as "Permissive for development", deploying these settings to production inherently negates cross-origin protections and leaves the application exposed to XSS data exfiltration and CSRF pivoting.

Remediation

Wrap the security module configuration in environment checks (e.g., using process.env.NODE_ENV) to ensure strict CSP and whitelisted CORS origins are initialized when in production context.

Potential Stored XSS via Direct DOM Injection

Medium

Location: app/components/service/Detail.vue

Description: Dynamic content retrieved via the /api/services/[slug] endpoint is injected directly into the DOM using the v-html directive (<div v-html="service.content"></div>). If administrative users, or a compromised staff account, store malicious payloads into a service object, it will execute contextually on the victim’s client side without further friction.

Remediation

Prior to rendering or storing HTML strings, sanitize the payload using a reliable DOM purifying library such as dompurify or vue-dompurify-html.

Absence of Content Rate Limiting on Uploads

Low

Location: server/api/applicants/[id]/documents.post.ts

Description: While file sizes are restricted to 500KB, there is no apparent threshold preventing a legitimate applicant from submitting thousands of individual documents back-to-back, establishing ground for resource exhaustion on the underlying filesystem.

Remediation

Implement an API rate limiting wrapper or leverage Nuxt-Security's rate limiting features specifically tied to file upload endpoints.

Architecture & Security Posture Notes

  • Role-Based Access Control (RBAC): The application utilizes robust hybrid RBAC in guards.ts and authentication middleware correctly, mitigating most IDOR (Insecure Direct Object Reference) attempts out-of-the-gate. Segment authorization acts accordingly based on session profiles.
  • Database Connectivity: Standard Prisma configuration is utilized securely with parameterized operations ($queryRaw\`...${variable}\` syntax is immune to traditional SQL injection vectors).
